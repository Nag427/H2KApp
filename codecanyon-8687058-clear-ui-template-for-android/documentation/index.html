Configuring

If you want to integrate ClearUI with your current application (instead of building your app in the ClearUI project). You should copy the following:

What you should copy if you want a quick start:

  • Every file in RES and all Java files in the .menu and .utils package + all java files that you're going to use from the main package (predefined layouts).
  • Clean up later with a tool like: https://code.google.com/p/android-unused-resources/, https://github.com/KeepSafe/android-resource-remover

What you should copy if you want to use everything in this kit, or the predefined layouts:

  • All the java files in the .menu & .utils packages
  • All the java files OR The java files of the layouts that you are planning to use, including Adapters, Parents, etc.
  • All drawables (keep them in the same folders), except the ones that start with ph_
  • All the layout files OR the layouts files that you're planning to use, including the layout_menu, layout_menu_item and layout_dialog xml files
  • All the content of the values folders (keep them in the same folders).
  • All the content of the anim folders.
  • All the content of the lib folders OR add the libraries manually.

What you should copy if you only want to use parts of this kit:

  • The content of the .menu package if you want to use the menu
  • The content of the .utils package if you want to use a custom view
  • All the XML content of the drawable folder (recommended) if you want to use the UI Elements
  • The shadow.9.png and title_bar_* files in the drawable-hdpi folder if you want to use the menu
  • The drawable-v14 content if you want to use the UI Elements
  • The (or a) menu_background.png file (in drawable-xhpdi) if you want to use the menu
  • The layout_menu and layout_menu_item files (in res/layout) if you want to use the menu
  • The layout_dialog.xml files (in res/layout) if you want to use the UI Elements
  • The attrs.xml file (in res/values) if you want to use the UI Elements
  • The colors.xml, file (in res/values)  (recommended) if you want to use the UI Elements
  • The content of the styles.xml (in res/values)  or at least the AppTheme and AppThemeTrans styles.
  • The content of dimens.xml (in res/values) 
  • The content  of styles.xml (in res/values-v19) 
  • All the content of the libs folder OR add the libraries manually.

After you've done this, you can start using your components. What is very important is you Theme Color. You can set it in your color.xml file (theme_color). This is the default color for the predefined layouts, and most UI Elements. Of course, you can multiple colors (we'll talk about that later in these docs), but your theme_color is the one you use most and configuring it will save you time.

The 3D Parralax Menu (I'll be calling it "the Menu" from here on). Will switch Fragments, but it should work with Activities to (you'll have to perform an Itent instead of ChangeFragment  but this will result in losing your Close() animation). Fire up MainActivity.java and have a look at private void setUpMenu() 

You'll see a few ways to customize the Menu here. You'll notice setHeaderView  which will create the FadingActionbar Effect. If you do not want to use this, you should customize the ResideMenu.java and remove associations with  headerView.

Initializing the Menu; 

resideMenu = new ResideMenu(this);
resideMenu.setBackground(R.drawable.menu_background);
resideMenu.attachToActivity(this);
resideMenu.setShadowVisible(true);
resideMenu.setHeaderView(findViewById(R.id.actionbar));
resideMenu.setMenuListener(menuListener);
 //valid scale factor is between 0.0f and 1.0f. leftmenu'width is 150dip. 
resideMenu.setScaleValue(0.6f);

itemHome     = new ResideMenuItem(this, R.drawable.ic_home,     "Home");
itemHome.setOnClickListener(this); //Your activity has to extend onClickListener
resideMenu.addMenuItem(itemHome);

Opening the Menu; 

//Call
resideMenu.openMenu(ResideMenu.DIRECTION_LEFT);
//To enable swiping override in your Activity
@Override
public boolean dispatchTouchEvent(MotionEvent ev) {
    return
    resideMenu.dispatchTouchEvent(ev);
}
That's really all there is to "Configure"

Customizing

You can customize the UI elements in a few ways: You can change the color, You can reuse them, You can add icons.

Changing the color

There are a 3 ways to change the Color(s) that an UI element appears in, the examples given are just "examples", I would always recommend at how it is done in our Elements Fragment:

1. Change the theme_color in colors.xml (and use a UIButton or UIButtonInverse style attribute). This will be applied to ALL UI Elements 

<Button
android:id="@+id/button1"         
android:layout_margin="3dp"         
android:layout_width="wrap_content"         
android:layout_height="wrap_content"         
android:text="@string/button_placeholder_text"         
style="@style/UIButton" />

<Button
android:id="@+id/button2"
android:layout_width="wrap_content"         
android:layout_height="wrap_content"
android:layout_margin="3dp"
android:text="@string/button_placeholder_text"         
style="@style/UIButtonInverse" />

2. Duplicate the drawables (&styles) and change the colors of the duplicates. (I would recommend this if you are using "2 kinds of layouts". E.g. if you would like to have a blue checkbox and a red checkbox you'd have to copy the following files: 

//You duplicate the following files, give them a new name (e.g. trottle_red.xml) and change
//the color references inside the duplicated file (e.g. change theme_color to green).
//    trottle_active.xml
//    trottle_xml

//now you should duplicate the "holder" which is checkbox.xml, and give it a new name (e.g.
//checkbox_red.xml). Now change the the drawable references inside the duplicated file (e.g. 
//change trottle to trottle_red

3. Change the color from code: 

//Changing the views background color, this will automatically work in both states press as
//well as non-press. (this is done in the 3th and 4th example button in the Elements fragment). This works best with transparent views.
 b3.getBackground().setColorFilter(getActivity().getResources().getColor(R.color.green), Mode.MULTIPLY);

        
 //Set the (pressed) text color of any view without having to modify and duplicate original layout files
 b3.setTextColor (new ColorStateList (
        		   new int [] [] {
        		      new int [] {android.R.attr.state_pressed},
        		      new int [] {}
        		   },
        		   new int [] {
        			getActivity().getResources().getColor(R.color.green),
        		      Color.WHITE
        		   }
        		));
        
 b4.setTextColor (new ColorStateList (
        		   new int [] [] {
        		      new int [] {android.R.attr.state_pressed},
        		      new int [] {}
        		   },
        		   new int [] {
        		      Color.WHITE,
        		      Color.BLACK
        		   }
        		));
// This will set a different text color on press than when not pressed. If you want to change a Views color in onpress (like
// you do with the text color). You should change the color filter in the ontouch method.

Reusing Items

You can let an LinearLayout act the same as a button. This is convenient if you want to create custom buttons. But this would also apply if you want to create a divider, or other new Element based on current Elements.

Check how we do it here: 

<LinearLayout
android:id="@+id/button2"
style="@style/UIButtonInverse"
android:layout_width="wrap_content"
android:layout_height="wrap_content"
android:layout_below="@+id/comment"
android:layout_marginLeft="5dp"
android:layout_marginTop="5dp"
android:clickable="true"
android:layout_toRightOf="@+id/button1" >

<ImageView
android:id="@+id/imageView1"
android:layout_width="30dp"
android:layout_height="25dp"
android:adjustViewBounds="true"
android:paddingBottom="5dp"
android:paddingTop="5dp"
android:tint="@color/theme_color"
android:src="@drawable/ic_conversation" />

<TextView
android:layout_width="wrap_content"
android:layout_height="wrap_content"
android:layout_marginRight="5dp"
android:text="@string/reply"
android:textColor="@drawable/button_selector_text" />

</LinearLayout>

Adding icons

In most cases, I would prefer adding icons to buttons using "Reusing Items". But in some cases, like a textEdit, it would be easier to just use a compoundDrawable. 

<EditText
android:id="@+id/editText1"
android:padding="5dp"
android:layout_margin="5dp"
android:background="@drawable/tab_default"
android:textColor="@color/theme_color"
android:layout_width="match_parent"
android:layout_height="wrap_content"
android:hint="@string/text_input_hint"
android:drawableLeft="@android:drawable/ic_menu_search"
android:ems="10" >
</EditText>

It is important to use different sized drawables, otherwise, you might end up with very large or small drawables.

Custom Views

Using custom Views is very easy. You just have to load a few schemas in your layout before you can use them. Let's have a look at how that is done: 

<ScrollView   xmlns:android="http://schemas.android.com/apk/res/android"
xmlns:app="http://schemas.android.com/apk/res-auto"
xmlns:tools="http://schemas.android.com/tools"
android:layout_width="match_parent"
tools:context="com.special.MenuActivity$ProfileFragment"
android:background="@android:color/white"
android:layout_height="match_parent">

As you can see, we need to load xmlns:app, xmlns:tools, and tools:context. tools:context will basically be the "path" to your activity or fragment (check how it is done above in case of a fragment).

I'm not going to show you how to use all Custom Views, since you can see how they are used in their respective layouts.

UIParallaxScroll
activity_detail.xml AND (requires java code if you want fading header) DetailActivity.java

UITabs
activity_detail_transition AND (if you want a listener example) TransitionDetailActivity.java

UISwipable List
fragment_list_item_transition.xml and fragment_list_transition.xml AND (requires java code for configuring) TransitionDetailAdapter.java (setting hidden view) and TransitionListFragment.java

Requires: 

setActionLayout(Hidden View)
setItemLayout(Front View)
setIgnoredViewHandler() //optionally, this will prevent opening the menu when sliding the list.

UICircularImage
I'm going to give an XML example for this one: 

<com.special.utils.UICircularImage
android:id="@+id/rounded_image"
android:layout_margin="5dp"
android:layout_width="75dp"
android:layout_height="75dp"
android:src="@drawable/ic_trashcan"
android:padding="10dp"
app:line_width="1dp"
app:padding="20dp"
app:line_color_press="@color/green"
app:line_color="@color/theme_color"/>

As you can see, when using a custom view, the tag you use is the "path" to the Java class (file). UICircularImage features a few custom classes. including line_width, padding (normal padding wont work), line_color_press and line_color. I think the names clarify what each attribute does. If you'd like to set a OnClickListener like you would normally do, for an example, check the TransitionDetailActivity

Error Solving
If you're busy integrating, you might run into some errors. Layout colors might not match, or infating custom views might cause trouble. I'd like to ask you to Google you error first. Most times, it can be solved easily.

Also, if partially copying and pasting files to your own project does not work. I would recommend stripping down my example layouts to the parts that you want to use, and use those as a base for you application. 

If you really get stuck, feel free to email me, or check our support community at sherdle.com/support. Please do not ask questions about customization (E.g. I would like to have the layout also do this and this, I would like to have this and this effect, I was customizing and got this problem/error...).
